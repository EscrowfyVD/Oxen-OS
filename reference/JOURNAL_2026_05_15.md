## Décisions notables

### Décisions structurelles (validées avec Vernon en début de sprint)

1. **Migration SQL manuelle au lieu de `prisma migrate dev`** — le `DATABASE_URL` local pointe vers Railway prod, donc `migrate dev` aurait pu reset la prod en cas de divergence de schema history. Migration SQL écrite à la main + `prisma generate`. La migration s'appliquera au prochain `migrate deploy` sur Railway.

2. **Branche** : travail sur `claude/xenodochial-borg-038829` (worktree). Pas de pull `main` upstream (2 commits behind) — Vernon gère le rebase/merge via PR plus tard.

3. **Docs** : PRD-002 créé from scratch (n'existait pas dans le repo malgré la mention dans le brief). PRD-001 mapping + OXEN_OS_ROADMAP_OVERVIEW skip (n'existent pas, hors scope).

### Décisions techniques (prises pendant l'implémentation)

4. **Consolidation Batch 2 + 3 dans un seul commit logique** — la déprécation du placeholder `trigify_intent_signal` (`isActive=false`) rend impossible un état intermédiaire fonctionnel entre "matching upgraded" et "signal mapping done". Séparer aurait introduit un downtime fonctionnel.

5. **Placeholder email synthétisé pour auto-create** — `CrmContact.email` est NOT NULL UNIQUE. Format `<linkedin-slug>@trigify.placeholder` choisi pour déterminisme (re-runs idempotents) et lisibilité dans le CRM.

6. **`vi.resetAllMocks()` au lieu de `vi.clearAllMocks()` dans `beforeEach`** — `clearAllMocks` ne purge pas la queue `mockResolvedValueOnce`, ce qui causait des leakages entre tests (vu 2 fois pendant le sprint avant correction).
